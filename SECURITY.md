# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this repository, please report it
privately instead of opening a public issue.

- Use the contact form on the website (mark your subject as `[SECURITY]`).
- Do not disclose the vulnerability publicly until it has been acknowledged
  and patched.

## Scope

Security issues in the deployed website or its build pipeline are in scope.
General feature requests and bug reports are not security issues — please use
the contact form for those.

## Hardening

This repository has branch protection, dependency vulnerability alerts, and
automated security fixes enabled.
